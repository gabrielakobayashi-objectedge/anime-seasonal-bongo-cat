# Anime Seasonal Tracker - Epic Breakdown

## Overview

This document provides the complete epic and story breakdown for the Anime Seasonal Tracker, decomposing the requirements into implementable stories. The project uses Playwright with the Page Object Model to scrape MyAnimeList's current seasonal anime listing, extract the top 5 rated anime, and save results as a JSON file with mock user reviews generated by Faker.

## Requirements Inventory

### Functional Requirements

FR1: Navigate to the MyAnimeList current season page, using automatic season detection (month → season mapping: Jan–Mar=Winter, Apr–Jun=Spring, Jul–Sep=Summer, Oct–Dec=Fall) to construct the correct URL.
FR2: Extract the top 5 anime entries from the current season, capturing title and rating/score.
FR3: Filter out any anime entry that does not have a rating/score (null, N/A, or empty).
FR4: Persist the filtered top-5 results (title, rating, mock_user_review) to a JSON output file.
FR5: Capture a screenshot of the #1 ranked anime entry after filtering.
FR6: Generate mock "User Review" data (via Faker) for each of the 5 anime entries and include it in the JSON output.
FR7: The JSON output structure must include at minimum: title, score, and mock_user_review per entry.

### NonFunctional Requirements

NFR1: The solution must use Playwright as the browser automation framework.
NFR2: The codebase must follow the Page Object Model (POM) architectural pattern.
NFR3: The Faker library must be used for all mock data generation.
NFR4: Code must be organized into at minimum three distinct units: PageManager, NavigationPage, and test spec.

### Additional Requirements

- PageManager: A central registry/factory that instantiates and exposes page objects.
- NavigationPage: A POM class encapsulating all MAL selectors, season-detection logic, and navigation behavior. Must construct the MAL seasonal URL dynamically based on current month/year.
- Test spec file: The orchestration entry point — drives the flow, writes JSON, captures screenshot, asserts outcomes.
- Output: A JSON file written to disk as the primary deliverable artifact.
- Screenshot: Saved to disk alongside the JSON output, targeting the #1 anime entry.

### UX Design Requirements

N/A — headless automation script, no user interface.

### FR Coverage Map

FR1 → Epic 1 — Season-aware MAL navigation (NavigationPage + PageManager)
FR2 → Epic 2 — Extract top 5 anime (title + score)
FR3 → Epic 2 — Filter entries missing a rating/score
FR4 → Epic 2 — Persist results to JSON file
FR5 → Epic 2 — Screenshot of the #1 anime entry
FR6 → Epic 2 — Faker mock user reviews per entry
FR7 → Epic 2 — JSON output structure: title, score, mock_user_review

## Epic 1: POM Foundation & Seasonal Navigation

A working Playwright project with proper POM architecture — functional `PageManager` and `NavigationPage` that automatically detect the current season and navigate to the correct MyAnimeList seasonal listing. After this epic the developer has a runnable, structured codebase ready to plug data logic into.

### Story 1.1: Project Setup & Environment Configuration

As a **developer**,
I want a properly initialized Playwright project with all required dependencies and folder structure,
So that I have a stable, organized foundation to build the scraper on.

**Acceptance Criteria:**

**Given** a fresh project directory
**When** the developer runs the install command
**Then** Playwright and Faker are installed as dependencies
**And** the folder structure exists: `pages/`, `tests/`, `output/`
**And** a `playwright.config.ts` (or `.js`) is present with a single Chromium browser configured
**And** a `.gitignore` excludes `node_modules/` and `output/`

### Story 1.2: NavigationPage — Season Detection & MAL Navigation

As a **developer**,
I want a `NavigationPage` POM class that auto-detects the current season and navigates to the correct MyAnimeList seasonal anime URL,
So that the scraper always targets the right listing without manual URL updates.

**Acceptance Criteria:**

**Given** the current date is known
**When** `NavigationPage.goToCurrentSeason()` is called
**Then** it derives the correct season from the current month (Jan–Mar=Winter, Apr–Jun=Spring, Jul–Sep=Summer, Oct–Dec=Fall)
**And** it navigates to `https://myanimelist.net/anime/season/{year}/{season}`
**And** the MAL seasonal listing page loads successfully (HTTP 200, main content visible)
**And** all MAL selectors used are encapsulated within `NavigationPage` (no selectors leak into the test spec)

### Story 1.3: PageManager — Page Object Registry

As a **developer**,
I want a `PageManager` class that acts as a central factory/registry for all page objects,
So that the test spec can access any page object through a single, consistent interface.

**Acceptance Criteria:**

**Given** a Playwright `Page` instance
**When** `new PageManager(page)` is instantiated
**Then** it exposes a `navigationPage` property that returns a `NavigationPage` instance
**And** `PageManager` does not contain any selector logic or direct DOM interaction
**And** adding a new page object requires only adding it to `PageManager` (no changes needed in the test spec)

## Epic 2: Anime Data Extraction, Enrichment & JSON Output

The complete data pipeline: scrape and filter the top 5 rated anime, generate Faker mock user reviews per entry, capture a screenshot of the #1 result, and persist the full dataset to a structured JSON file — all orchestrated through the test spec file.

### Story 2.1: Scrape & Filter Top 5 Rated Anime

As a **developer**,
I want `NavigationPage` to expose a method that extracts anime entries from the seasonal listing and returns only the top 5 with a valid score,
So that the scraper collects clean, ranked data ready for enrichment.

**Acceptance Criteria:**

**Given** the MAL seasonal page has loaded
**When** `NavigationPage.getTopRatedAnime()` is called
**Then** it reads all anime entries visible on the page
**And** it filters out any entry where the score is missing, "N/A", or not a number
**And** it returns the top 5 entries sorted by score (descending), each containing `title` and `score`
**And** if fewer than 5 rated entries exist on the page, it returns however many are available without throwing

### Story 2.2: Generate Faker Mock User Reviews

As a **developer**,
I want a utility function that generates a mock user review object for an anime entry using Faker,
So that each JSON result is enriched with realistic-looking (but synthetic) review data.

**Acceptance Criteria:**

**Given** an anime entry with a `title`
**When** `generateMockReview(title)` is called
**Then** it returns an object containing at minimum: `reviewer_name`, `rating` (1–10), and `comment`
**And** all values are generated by Faker (no hardcoded strings)
**And** calling the function twice for the same title produces different output (non-deterministic)
**And** the function is importable as a standalone utility (not embedded in a page object or test spec)

### Story 2.3: Screenshot Capture & JSON Output

As a **developer**,
I want the scraper to screenshot the #1 anime card and write the full enriched dataset to a JSON file,
So that the run produces two concrete, inspectable artifacts on disk.

**Acceptance Criteria:**

**Given** a filtered, ranked list of top 5 anime with mock reviews attached
**When** the output step executes
**Then** a screenshot of the #1 anime card element is saved to `output/top-anime-screenshot.png`
**And** a JSON file is written to `output/anime-results.json` with an array of 5 objects, each containing `title`, `score`, and `mock_user_review`
**And** `mock_user_review` is the full object from Story 2.2 (not a flat string)
**And** the `output/` directory is created automatically if it does not exist

### Story 2.4: Test Spec — Full Pipeline Orchestration

As a **developer**,
I want a Playwright test spec that wires together all components into a single executable run,
So that one command produces the screenshot and JSON output end-to-end.

**Acceptance Criteria:**

**Given** the project dependencies are installed
**When** `npx playwright test` is run
**Then** the test instantiates `PageManager`, calls `navigationPage.goToCurrentSeason()`, retrieves the top 5 via `getTopRatedAnime()`, generates mock reviews, captures the screenshot, and writes `anime-results.json`
**And** the test spec contains no selector strings (all selectors live in `NavigationPage`)
**And** the test passes with a green result when `output/anime-results.json` exists and contains 5 entries
**And** the test fails with a descriptive message if fewer than 1 rated anime is found on the page
